# CMake Playground

This is a simple C++ project to test the capabilities of CMake & CPack in combination with various generators. It's aim is to provide a template for a (probably not) fully automated C++ build environment that:

- builds on various platforms
- generates installers for various platforms
- has a fully automated release process
- runs static code analysis
- validates code style

The current project setup contains a simple C++ commons library. Furthermore there are test projects and an executable that uses the libraries. The C++ source code is mostly generated by AI because it is not the main focus of this project. The main focus is the project setup and structure.

## Disclaimer

This project is not copmlete and will probably contain errors. It is based on my experience with these tools and is not guaranteed to follow every best practice. The repository is intended as a personal notebook that may also be helpful to others. Feedback and improvement suggestions are welcome.

## Features

The available features will be described and explained in the following sections.

### CMake

Most of the project rely on internal CMake and CPack functionalities.

In general each target is created in a separate directory and a separate list file. The libraries are declared using [`add_library`](https://cmake.org/cmake/help/latest/command/add_library.html) whereas executables use [`add_executable`](https://cmake.org/cmake/help/latest/command/add_executable.html).

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/cli/CMakeLists.txt#L5

To make sure targets are found even when using namespaces aliases are added for libraries as well.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/math/CMakeLists.txt#L3-L4

To specify the source files the function [`target_sources`](https://cmake.org/cmake/help/latest/command/target_sources.html) is used. Within this function you can distinguish between internal private files (mostly implementation files or headers that should not be available in targets using this library) and external public files (mostly the API header files).

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/math/CMakeLists.txt#L6-L15

The directories containing the header files will then be made available using [`target_include_directories`](https://cmake.org/cmake/help/latest/command/target_include_directories.html). To make sure the headers are also found when using installed target the install include directory will be added as well.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/math/CMakeLists.txt#L17-L19

Export headers can also be generated by CMake using the [`generate_export_header`](https://cmake.org/cmake/help/latest/module/GenerateExportHeader.html) function and including resulting files in the public headers installation directory.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/math/CMakeLists.txt#L23-L30

To detect breaking changes in API the compatiblity will be set using different properties on each target.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/math/CMakeLists.txt#L32-L39

Finally all generated and created files will be installed to their corresponding locations. [`GNUInstallDirs`](https://cmake.org/cmake/help/latest/module/GNUInstallDirs.html) will be used as default locations.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/math/CMakeLists.txt#L41-L49

To generate CMake target files which can be used by developers using this library to reference it the [`install(EXPORT ...)`](https://cmake.org/cmake/help/latest/command/install.html#export) function is being used.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/math/CMakeLists.txt#L41-L42

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/math/CMakeLists.txt#L51-L56

#### Version from git tags

Versioning of the CMake project will be automatically done by reading and parsing all Git tags. Versioning scheme follows semantic versioning and parsing is implemented in a [separate file](./cmake/GenerateVersion.cmake). It makes the result available in four variables that contain each part of the semantic release version.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/cmake/GenerateVersion.cmake#L28-L31

These will be used in each target to set the version compatability discussed before and in the root project definition which propagates it's version to all child projects.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/CMakeLists.txt#L5-L6

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/math/CMakeLists.txt#L32-L39

#### SBOM

An SPDX SBOM file will automatically be generated and installed by using CMake's experimental [`export(SBOM ...)`](https://cmake.org/cmake/help/latest/command/export.html#sbom) function.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/CMakeLists.txt#L80-L93

#### CPack

For packing build results various CPack generators will be used. This is done only once in the root CMakeLists.txt file. Currently there are no special cases handled and everything will be packed as it is. But the generators are capable to generate e.g. fancy Windows MSI installers. Check the [CPack generators documentation](https://cmake.org/cmake/help/latest/manual/cpack-generators.7.html) for details.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/CMakeLists.txt#L95-L137

### Tests

For testing the GTest library is used which is installed with Conan. The test projects are within their corresponding projects and should test the API or internal functions.

https://github.com/o-lukas/cmake-playground/blob/f1bc234a9bef62ffec6872e76bb87318db47a9b2/src/string/test/src/StringUtilTest.cpp#L1-L8

Tests are made available to CTest using gtest_discover_tests function so they can easily be executed within the CMake environment.
